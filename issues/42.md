## Description
Enhance the git-lfs-file-locker extension to automatically log each file lock operation to a centralized log file. This log should record when a file is locked, on which branch, and by whom.

## Use Case
When a file is locked via the extension, store this information in a log file (e.g., `lock-log.md` or `lock-log.json`) so that team members can see:
- Which files are currently locked
- Which branch the lock was performed on
- When the lock was created
- Who locked the file

This helps with transparency and coordination when multiple developers are working with Git LFS locks.

## Proposed Implementation
- Add the current branch name to the lock log entry when a file is locked
- Store logs in a configurable log file (default: `.lfs-locks.log` or similar)
- Update the log entry when a file is unlocked
- Consider supporting both human-readable (Markdown) and machine-readable (JSON) formats

## Additional Context
This would complement the existing lock/unlock functionality by providing a persistent, auditable record of all lock operations in the repository.
